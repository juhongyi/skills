STRICTLY APPLY THE MINIMAL CHANGE PRINCIPLE: make only the changes necessary to complete the requested work, and avoid unrelated or nonessential changes, broad rewrites, and scope expansion.

For any ambiguity, conflict, or user-dependent choice in the task, plan, or execution, keep asking the user until it is fully resolved.  
When drafting or updating plans, write each step as a positive action statement that describes only the work to be done.

Break all work into the smallest practical units, and keep each commit focused on one clear change.  
When a test environment is available, do not commit unless the full test suite passes.
